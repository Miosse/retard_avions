Choix des features à utiliser pour notre modèle¶

Donnée Cible¶

Tout d'abord rappelons que la donn√©e que nous cherchons √† pr√©dire est le d√©lai constat√© pour le vol.

Cette information correspond tout simplement √† la donn√©e ARR_DELAY, c'est √† dire le d√©lai √† l'arriv√©e.

Features Catégorielles¶

  • Nous avons vu dans la partie exploratoire, que l'heure d'arriv√©e d√©pend fortement de la date (FL_DATE, aussi d√©compos√©e en MONTH / DAY_OF_MONTH)

  • Nous devons aussi prendre en compte les points d'arriv√©e et de d√©part pour mod√©liser au mieux nos r√©sultats

    Ce sont les features : 'ORIGIN_AIRPORT_SEQ_ID' et 'DEST_AIRPORT_SEQ_ID',

  • L'heure de d√©part a aussi une forte influence.

    Ce sont les features : 'ARR_HOUR' et 'DEP_HOUR'

  • Le jour de la semaine a aussi son importance :

    Nous prenons : 'DAY_OF_WEEK'

  • Nous ne prenons pas les donn√©es li√©es au D√©lai qui correspondent √† du DataLeakage : √† savoir que ces donn√©es sont des donn√©es connues √† post√©riori : donc ne peuvent se trouver dans les informations √† utiliser pour la pr√©diction

Features Numériques¶

  • Nous allons de plus utiliser la Distance entre 2 a√©roports comme features num√©rique

La mesure des données.¶

Pour mesure la performance de notre mod√®le nous allons utiliser le RMSE (ou Root-Mean-Square-Error)
